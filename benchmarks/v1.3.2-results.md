# Benchmark Results — v1.3.2+ Performance Improvements

Captured against `benchmarks/benchmark-baseline.json` (recorded 2026-04-10 at v1.3.0).

## Methodology

- Local capture via `scripts/run-bench-snapshot.mjs` on dev hardware
- 1 warmup render, then 3 measured renders per corpus, average reported
- Same corpora manifest (`src/benchmarks/corpora.ts`) used by the regression guard
- Compares against `stages.totalMs` from baseline

## Results (v1.3.4)

| Corpus                  | Baseline totalMs | v1.3.4 avgMs | Delta   |
|-------------------------|------------------|--------------|---------|
| rich-text-mixed-spans   | 273.7            | 156.4        | -42.8%  |
| table-stress            | 515.9            | 247.8        | -52.0%  |
| cjk-layout              | 195.7            | 141.5        | -27.7%  |
| rtl-layout              | 294.9            | 167.5        | -43.2%  |
| punctuation-heavy       | 306.6            | 149.4        | -51.3%  |
| invoice-showcase        | 247.5            | 149.6        | -39.6%  |
| report-showcase         | 341.3            | 215.0        | -37.0%  |

**Geometric-mean speedup: ~1.66x** across the seven core corpora.

## Drivers

Improvements landed across v1.3.1 → v1.3.4:

- **v1.3.2 — DNS dedup** for remote asset fetches (one resolve per host per render)
- **v1.3.2 — Parallel raster** for image elements (was sequential)
- **v1.3.2 — Word-width cache** in measure layer (avoids re-measuring repeated tokens)
- **v1.3.3+** — incidental cleanups (vendored pretext patch updates, smaller hot paths)

The largest wins appear in text-heavy corpora (`punctuation-heavy`, `table-stress`,
`rtl-layout`), consistent with the word-width cache being the dominant lever.

## Caveats

- Numbers are dev-hardware (Windows, Node 24.13). CI runners are slower and noisier;
  the regression guard in `test/benchmark-baseline.test.ts` uses 3x slack + 5s floor
  precisely because of this.
- Baseline was captured on the same machine class but in April 2026; absolute numbers
  may shift with Node/V8 version drift independent of pretext-pdf changes.
- Geometric-mean speedup is an order-of-magnitude indicator, not a guarantee.

## Reproducing

```bash
npx tsx scripts/run-bench-snapshot.mjs
```
