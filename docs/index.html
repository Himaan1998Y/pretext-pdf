PDF generation
the way LLMs think

A PdfDocument is plain JSON. LLMs emit it in one shot — no codegen, no headless browser, no Chromium. Professional typography. MCP server included.

Live Demo GitHub npm
invoice.ts 
import { generate } from 'pretext-pdf';

const pdf = await generate({
  template: 'invoice',
  content: {
    from: 'Acme Corp',
    to: 'Client Ltd',
    items: [
      {
        name: 'Consulting',
        qty: 10,
        rate: 150
      }
    ],
    currency: 'USD'
  }
});

// Buffer — write to disk or stream
fs.writeFileSync(
  'invoice.pdf', pdf
);

Features

Everything a modern PDF stack needs

No browser runtime. No bloated dependencies. Just a clean TypeScript library that runs anywhere JavaScript does.

⚡

No Chromium required

Puppeteer and playwright ship a 300 MB browser binary. pretext-pdf is a pure JS PDF renderer — works in serverless functions, edge workers, and containers without any native dependencies.

# pdfmake / puppeteer
node_modules size: ~350MB cold start: ~4s

# pretext-pdf
node_modules size: ~12MB cold start: ~180ms
🤖

LLM-native document schema

A PdfDocument is plain JSON. Any language model can emit it in one shot. No template strings, no function calls, no AST — just structured data.

🔤

Professional typography

Kerning, hyphenation, justified text, RTL (Hebrew, Arabic) and CJK (Chinese, Japanese, Korean) out of the box.

📊

Vega-Lite charts

Declare a Vega-Lite spec inline — pretext-pdf renders it to SVG and embeds it natively. Data visualization as a first-class citizen.

📋

Tables & complex layouts

Colspan, rowspan, nested tables, per-cell styling, sticky headers — all from JSON without a single line of layout code.

Every output primitive you need

From QR codes to barcodes, Markdown blocks to raw SVG, images to invoices — the schema covers the full surface area of real-world documents.

Tables with colspan/rowspan Vega-Lite charts QR codes Barcodes Images (JPEG, PNG, SVG) Inline SVG Markdown input Invoice template Report template GST invoices RTL & CJK text Custom fonts Headers & footers Page numbers Watermarks Multi-column layout
How it works

JSON in. PDF out. That simple.

Three steps is all it takes — whether you're building a server-side report or wiring an LLM to a document workflow.

STEP 01

Describe your document

Write (or have an LLM generate) a plain PdfDocument JSON object. No function calls, no template literals — just a structured description of your PDF.

STEP 02

Call generate()

Pass the document object to the generate() function. It runs synchronously in Node, Deno, Bun, or any edge runtime — returns a Buffer.

STEP 03

Stream, save, or serve

Write the buffer to disk, pipe it as an HTTP response, upload to S3, or hand it to an email attachment. It's just bytes.

MCP Server

Add PDF generation to any AI client

pretext-pdf-mcp is an MCP server that exposes the PDF engine as a tool. Claude, Cursor, Windsurf — one config, any client.

Install the library
$ npm install pretext-pdf
Or install the MCP server globally
$ npm install -g pretext-pdf-mcp
Or run directly with npx (no install)
$ npx -y pretext-pdf-mcp
Works with
Claude Desktop Cursor Windsurf Any MCP Client
JSON mcp-config.json 
{
  "mcpServers": {
    "pretext-pdf": {
      "command": "npx",
      "args": [
        "-y",
        "pretext-pdf-mcp"
      ]
    }
  }
}

Once connected, ask Claude: "Generate a PDF invoice for my client" — it will emit the JSON schema and call the tool to produce a real PDF.
